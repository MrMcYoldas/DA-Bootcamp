# 2026-03-18 – Lesson / Topic If-Statements

## Topic

Today I learned about if-statements and how they control the flow of a Python program. The main idea is that code only runs when a condition is true. This includes understanding `if`, `elif`, and `else`, as well as working with boolean values (`True` / `False`), comparison operators, and user input using `input()`.

## Key Concepts

If-statements allow me to define conditions that evaluate to either True or False. Based on that, Python decides whether to execute a block of code. The syntax always ends with a colon `:` and the block is defined by indentation. I also learned comparison operators like `==`, `!=`, `<`, `>`, `<=`, `>=`, and logical operators such as `and`, `or`, and `not`. Another important concept is that `input()` always returns a string, so I need to convert it using `int()` when working with numbers.

## What I Learned

I now understand that if-statements are the foundation of controlling program logic. Only the first true condition in an if/elif/else chain is executed, which is important to remember. I also learned how to structure conditions properly and how indentation defines the code block. Additionally, I saw how nested if-statements can often be simplified using logical operators like `and` and `or`, which makes the code cleaner and easier to read.

## What I Found Interesting

The topic was very interesting and overall quite straightforward. The logic behind conditions feels intuitive once understood. Especially the idea of combining conditions instead of nesting them made things much clearer and more efficient.

## Challenges

I found it confusing to write everything inside one code cell, especially when multiple conditions are involved. Also, getting used to indentation and the structure of if-statements still takes some practice. There are many new terms, so building muscle memory will be important.

## Reflection

This lesson was very helpful and I learned a lot. The concepts themselves are not too difficult, but applying them cleanly in code requires practice. I noticed that I sometimes overcomplicate things, so keeping solutions simple is key. Writing everything in one cell is still a bit tricky, but I understand the logic much better now.

## Key Takeaways

If-statements control the flow of a program based on conditions. Only one condition runs in an if/elif/else chain. Input from users is always a string and needs conversion when working with numbers. Logical operators like `and`, `or`, and `not` help simplify conditions and avoid unnecessary nesting.
